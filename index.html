Role Management

Granular access control,
effortlessly managed.

Create unlimited roles with fine-grained privileges. Check permissions in controllers, middleware, Blade templates, or anywhere in your code with intuitive helpers. Supports wildcard matching.

$user->hasRole('admin')
$user->hasRoles(['admin','editor'])
$user->hasPrivileges([...])
$user->can('reports.run')
UserController.php 
// Check if user has admin role
if ($user->hasRole('admin')) {
    // Grant access
}

// Check multiple roles (user must have ALL)
if ($user->hasRoles(['admin', 'super-admin'])) {
    // User has both roles
}

// Check specific privilege via Gate
if ($user->can('reports.run')) {
    // User can run reports
}

// Require all listed privileges
$hasAccess = $user->hasPrivileges([
    'reports.run',
    'billing.view'
]);
Powerful Middleware

Protect routes
with precision.

Eight purpose-built middleware aliases for enforcing role and privilege checks directly on your routes. Require all or any abilities with flexible syntax.

role:admin
roles:editor,admin
privilege:reports.run
privileges:a,b
ability:admin,edit
abilities:x,y
routes/api.php 
use Illuminate\Support\Facades\Route;

// Require admin role
Route::middleware(['auth:sanctum', 'role:admin'])
    ->get('dashboard', DashboardController::class);

// Allow editor OR admin
Route::middleware(['auth:sanctum', 'roles:editor,admin'])
    ->post('articles', ArticleController::class);

// Require ALL listed privileges
Route::middleware('privilege:reports.run,billing.view')
    ->get('reports', ReportsController::class);

// Audit sensitive routes
Route::middleware(['auth:sanctum', 'role:admin', 'tyro.log'])
    ->delete('users/{user}', [UserController::class, 'destroy']);

Role Methods

Check roles
with precision.

Verify single or multiple roles with intuitive methods. Wildcard support lets you match patterns. All role checks are cached for optimal performance.

hasRole() Check single role
hasRoles() Require ALL roles
hasAnyRole() Match ANY role
tyroRoleSlugs() Get all role slugs
Role Checks 
// Check if user has a specific role
if ($user->hasRole('admin')) {
    // User is an admin
}

// Wildcard matching
if ($user->hasRole('*-admin')) {
    // Matches super-admin, content-admin, etc.
}

// Check multiple roles (user must have ALL)
if ($user->hasRoles(['admin', 'billing-manager'])) {
    // User has both roles
}

// Check if user has ANY of the roles
if ($user->hasAnyRole('editor', 'author', 'contributor')) {
    // User has at least one
}

// Get all role slugs (cached)
$roles = $user->tyroRoleSlugs();
// Returns: ['admin', 'editor']
Privilege Methods

Fine-grained
permissions.

Privileges are inherited through roles. Check specific abilities like reports.run or users.delete. The unified can() method checks privileges first, then roles, then falls back to Laravel Gate.

hasPrivilege() Check single privilege
hasPrivileges() Require ALL privileges
can() Unified check (Gate)
tyroPrivilegeSlugs() Get all privilege slugs
Privilege Checks 
// Check single privilege via Gate integration
if ($user->can('reports.run')) {
    // User can run reports
}

// Direct privilege check
if ($user->hasPrivilege('users.delete')) {
    // User has delete-users privilege
}

// Require ALL privileges
if ($user->hasPrivileges(['reports.run', 'billing.view'])) {
    // User has BOTH privileges
}

// Get all privilege slugs (cached)
$privileges = $user->tyroPrivilegeSlugs();
// Returns: ['reports.run', 'billing.view', ...]

// can() checks privilege, then role, then Gate
if ($user->can('admin')) {
    // Checks if user has 'admin' privilege OR 'admin' role
}
Programmatic Control

Manage roles &
privileges at runtime.

Assign and remove roles dynamically. Attach privileges to roles. Build admin panels, self-service portals, or automated permission workflows with the full Role and Privilege model APIs.

Assign/remove roles without detaching existing ones
Attach/detach privileges to roles
Role model has hasPrivilege() and hasPrivileges()
Runtime Management 
use HasinHayder\Tyro\Models\Role;
use HasinHayder\Tyro\Models\Privilege;

// Assign role to user
$role = Role::where('slug', 'editor')->first();
$user->assignRole($role);

// Remove role from user
$user->removeRole($role);

// Attach privilege to role
$privilege = Privilege::where('slug', 'reports.export')->first();
$role->privileges()->attach($privilege->id);

// Detach privilege from role
$role->privileges()->detach($privilege->id);

// Check if role has a privilege
if ($role->hasPrivilege('reports.run')) {
    // Role has this privilege
}

// Check if role has ALL privileges
if ($role->hasPrivileges(['reports.run', 'billing.view'])) {
    // Role has both
}

Sanctum Integration

Security that
never sleeps.

Sanctum tokens automatically mirror role and privilege slugs as abilities. Suspension workflows revoke tokens instantly. Emergency token rotation is one command away.

Token abilities include role and privilege slugs
Suspension instantly revokes all active tokens
Per-user slug caching for optimal performance
AuthController.php 
// Suspend a user (revokes all tokens automatically)
$user->suspend('Pending account review');

// Check suspension status
if ($user->isSuspended()) {
    $reason = $user->getSuspensionReason();
}

// Unsuspend and restore access
$user->unsuspend();

// Emergency token rotation
// php artisan tyro:auth-logout-all-users --force
