Core API

Elegant Role & Privilege API

Check permissions anywhere in your application with a clean, fluent, and highly performant Laravel-native API.

  • Native Gate integration with $user->can()
  • Cached real-time privilege checking across roles
  • Instant user suspension and token revocation
  • Zero queries for repeated authorization checks
ReportController.php 
if ($request->user()->can('reports.run')) {
    // User has the privilege through their roles
}

if ($request->user()->hasRole('admin')) {
    // Check specific role
}

if ($request->user()->hasPrivileges(['billing.view', 'reports.run'])) {
    // Require multiple privileges
}

// Suspend a user and revoke all tokens
$user->suspend('Policy violation');

// Check suspension status
if ($user->isSuspended()) {
    return response()->json([
        'error' => $user->getSuspensionReason()
    ], 423);
}
AdminController.php 
// Suspend a user with reason
$user->suspend('Policy violation');

// All tokens are automatically revoked
// User cannot access protected routes

// Check suspension status
if ($user->isSuspended()) {
    $reason = $user->getSuspensionReason();
    return response()->json([
        'error' => 'Account suspended',
        'reason' => $reason
    ], 423);
}

// Unsuspend when ready
$user->unsuspend();

// CLI: php artisan tyro:user-suspend --user=1
// CLI: php artisan tyro:user-unsuspend --user=1

Middleware

Bulletproof Route Protection

Lock down your API endpoints and web routes instantly using powerful built-in middleware for roles, privileges, and token abilities.

Role-Based Protection

Restrict routes to users with specific roles using role: for a single role or roles: for multiple roles.

routes/api.php 
// Require exactly one role
Route::middleware(['auth:sanctum', 'role:admin'])->group(function () {
    Route::get('/admin/dashboard', [AdminController::class, 'dashboard']);
});

// Require ANY of the roles
Route::middleware(['auth:sanctum', 'roles:admin,editor'])->group(function () {
    Route::post('/content', [ContentController::class, 'store']);
});

// Per-route middleware
Route::get('/reports', [ReportController::class, 'index'])
    ->middleware(['auth:sanctum', 'role:analyst']);

Privilege-Based Protection

Check for specific privileges using privilege: or privileges: middleware.

routes/api.php 
// Require one specific privilege
Route::middleware(['auth:sanctum', 'privilege:reports.run'])
    ->get('/reports', [ReportController::class, 'index']);

// Require ANY of these privileges
Route::middleware(['auth:sanctum', 'privileges:billing.view,reports.run'])
    ->group(function () {
        Route::get('/dashboard/analytics', [AnalyticsController::class, 'index']);
    });

// Combine with role middleware
Route::middleware(['auth:sanctum', 'role:admin', 'privilege:system.manage'])
    ->post('/system/reset', [SystemController::class, 'reset']);

Sanctum Token Ability Checks

Tyro tokens automatically embed role and privilege slugs as Sanctum abilities. Check them directly via ability: middleware.

routes/api.php 
// Check token abilities directly
Route::middleware(['auth:sanctum', 'ability:admin'])
    ->get('/admin', [AdminController::class, 'index']);

// Require multiple abilities
Route::middleware(['auth:sanctum', 'abilities:admin,reports.run'])
    ->get('/admin/reports', [AdminReportController::class, 'index']);

// Manual token ability check in controller
if ($request->user()->tokenCan('billing.view')) {
    // Token has billing.view ability
}

All middleware automatically returns 403 Forbidden for unauthorized requests and 401 Unauthenticated for missing or invalid tokens.

Template Engine

Semantic Blade Directives

Keep your views clean. Conditionally render UI components based on user permissions with expressive, easy-to-read tags.

Blade Template 
{{-- Check if user has a specific ability via Gate --}}
@userCan('edit-posts')
    <button class="btn btn-primary">
        Edit Post
    </button>
@enduserCan

Hooks into Laravel's native Gate. Since Tyro registers all privilege slugs as Gate abilities, this works flawlessly with privilege names too.

Blade Template 
{{-- Show content only for admins --}}
@hasRole('admin')
    <div class="admin-panel">
        <h2>Admin Dashboard</h2>
        <a href="/admin/settings">Settings</a>
    </div>
@endhasRole
Blade Template 
{{-- Check if user has ANY of the roles --}}
@hasAnyRole('admin', 'editor', 'moderator')
    <div class="management-tools">
        <h3>Management Tools</h3>
        <a href="/manage">Open Dashboard</a>
    </div>
@endhasAnyRole
Blade Template 
{{-- Require ALL specified roles --}}
@hasAllRoles('admin', 'super-admin')
    <button class="btn-danger">
        System Configuration
    </button>
@endhasAllRoles
Blade Template 
{{-- Check for a specific privilege --}}
@hasPrivilege('delete-users')
    <button class="btn-danger" onclick="deleteUser()">
        Delete User
    </button>
@endhasPrivilege
Blade Template 
{{-- Check for ANY of the privileges --}}
@hasAnyPrivilege('edit-posts', 'delete-posts', 'publish-posts')
    <div class="post-actions">
        <h4>Post Management</h4>
        @hasPrivilege('edit-posts')
            <button>Edit</button>
        @endhasPrivilege
        @hasPrivilege('delete-posts')
            <button>Delete</button>
        @endhasPrivilege
        @hasPrivilege('publish-posts')
            <button>Publish</button>
        @endhasPrivilege
    </div>
@endhasAnyPrivilege
Blade Template 
{{-- Require ALL specified privileges --}}
@hasAllPrivileges('create-invoices', 'approve-invoices')
    <button class="btn btn-success">
        Create and Approve Invoice
    </button>
@endhasAllPrivileges

@hasAllPrivileges('view-reports', 'export-reports')
    <div class="reports-section">
        <a href="/reports">View Reports</a>
        <button onclick="exportReport()">
            Export to PDF
        </button>
    </div>
@endhasAllPrivileges

Frequently Asked Questions

Everything you need to know about Tyro and how it works.

Tyro is the ultimate Authentication, Authorization, and Role & Privilege Management solution for Laravel 12. It provides complete RBAC, user suspension workflows, 40+ CLI commands, 7 Blade directives (@userCan, @hasRole, @hasAnyRole, @hasAllRoles, @hasPrivilege, @hasAnyPrivilege, @hasAllPrivileges), middleware for route protection, and optional REST API endpoints. It works for APIs, traditional web apps, and hybrid applications—saving you weeks of development time.
Tyro requires PHP 8.2 or higher, Laravel 12.0 or higher, and Laravel Sanctum 4.0 or higher. It's designed to work seamlessly with fresh Laravel installations and can be integrated into existing projects.
Installation is simple: run composer require hasinhayder/tyro followed by php artisan tyro:install. The install command automatically sets up Sanctum, runs migrations, seeds default roles and privileges, and prepares your User model. That's it—your API is ready!
Tyro ships with 40+ artisan commands for managing users, roles, privileges, tokens, and more. Key commands include: tyro:user-create, tyro:role-assign, tyro:user-suspend, tyro:privilege-create, tyro:privilege-attach, tyro:user-token, and tyro:auth-logout-all-users. Every command supports non-interactive flags for automation.
Tyro uses a flexible role-privilege system. Roles (like "admin", "editor") can have multiple privileges (like "reports.run", "billing.view"). Users are assigned roles, and they inherit all privileges from those roles. You can check permissions using $user->can('privilege.name'), $user->hasRole('role-slug'), or middleware like ability:admin,reports.run. Sanctum tokens automatically include all role and privilege slugs as abilities.
Absolutely! Publish the config file with php artisan vendor:publish --tag=tyro-config to customize route prefixes, middleware, table names, default roles, and more. You can also publish migrations and disable built-in routes or commands via environment variables. Tyro is designed to be extensible without friction.
Use php artisan tyro:user-suspend --user=email@example.com --reason="Policy violation" or call $user->suspend('reason') in your code. Suspension automatically revokes all Sanctum tokens and prevents login. To lift a suspension, use php artisan tyro:user-unsuspend or $user->unsuspend(). Check suspension status with $user->isSuspended().
Yes! Tyro is battle-tested and ships with the same security hardening, middleware stack, and authorization logic used in production applications. It includes comprehensive tests, factories, seeders, and follows Laravel best practices. You can disable commands in production via TYRO_DISABLE_COMMANDS=true for additional security.
Yes! Tyro includes comprehensive audit logging that tracks all administrative actions—including role assignments, user suspensions, privilege changes, and role/privilege creation/deletion. Each log entry captures who performed the action, when it happened, IP address, user agent, and before/after values. You can view logs via php artisan tyro:audit-list, purge old logs with php artisan tyro:audit-purge, or access them via the REST API. Audit trails are essential for compliance and forensic analysis.
When updating Tyro from an older version, you must run migrations to apply any new database changes. After running composer update hasinhayder/tyro, immediately run php artisan migrate (or php artisan migrate --path=vendor/hasinhayder/tyro/database/migrations to run only Tyro migrations). Skipping migrations may result in missing database tables, audit logs not recording, and other unexpected behavior.
